Motivation

Scenario

  • Model Merging: Find optimal $\alpha$ on a small proxy → transfer to the large target. Save large-model tuning cost.
  • Quantization: Compress LLMs (bf16 → INT4/Q2); quantized model acts as proxy. Estimate quality degradation without expensive benchmarks.
  • Continual Learning: Fine-tune on Task A, use fine-tuned model as proxy. Estimate Task B forgetting before full re-evaluation.

Save Compute

Evaluate a cheap proxy instead of running full benchmarks on the expensive target model.

Estimate Performance

Predict the target's performance from the proxy's geometry — without ground-truth labels.

The Core Problem

There is a lack of an analytical/diagnosis formulation to explain these empirical results. We rely on heuristics and expensive evaluations.

Idea

Proxy Model and Target Model

  • Target ($T$): The expensive, large, or full-precision model we care about.
  • Proxy ($P$): The cheap, small, or compressed model we actually compute on.

Is the Proxy Model ($P$) good enough to estimate the performance of the Target Model ($T$)?

Explainability

We don't just want a black-box predictor. We need a mathematical framework that provides Explainability.

If the Proxy fails to track the Target, we need to mathematically isolate why.

4 / 24

Requirement

1. Explainability

The formulation must decouple different sources of errors (e.g., feature mismatch vs. classifier mismatch) so we can diagnose failures.

2. Computable

The metrics must be rigorously calculable from empirical data, without requiring infinite samples or unobservable ground truths.

3. Computational Efficiency

Evaluating the proxy's validity must be significantly cheaper than fully evaluating or training the target model.

4. Robustness

The theory must hold across different architectures, dimensionalities, and varying levels of geometric distortion.

Formulation

Methodology Choices

Analyze optimal point ($\alpha^*$)

Comparing parameters requires extremely strong assumptions because different models live in disjoint, non-comparable spaces.

Analyze Empirical Risk ($\Delta \mathcal{R}$)

We analyze the gap: $\Delta \mathcal{R} = |\mathcal{R}_T - \mathcal{R}_P|$

Why? Risk is a unified scalar. If we can prove the empirical risk landscapes are geometrically bound to each other, their optimal points must be functionally close.

Modern Models: Feature Representation & Linear Projection

We express any model as a composition: $f(x) = h(\phi(x))$, where $\phi(x)$ is the backbone and $h$ is the linear head. In matrix form over a dataset of $n$ samples:

$$\underbrace{F(X)}_{n \times c} \;=\; \underbrace{\phi(X)}_{n \times d} \;\cdot\; \underbrace{H}_{d \times c}$$
  • $Z = \phi(X) \in \mathbb{R}^{n \times d}$: feature matrix from the backbone ($n$ samples, $d$ dimensions).
  • $H \in \mathbb{R}^{d \times c}$: linear projection head ($c$ = number of classes).

Math: The Derivation Logic

1. Starting Point

Bound the risk gap, expanded by definition:

$$|\mathcal{R}_T - \mathcal{R}_P| = |\mathcal{R}_T(Z_T) - \mathcal{R}_P(Z_P)| = \left|\frac{1}{n}\sum_i \ell(z_i^T H_T,\, y_i) - \frac{1}{n}\sum_i \ell(z_i^P H_P,\, y_i)\right|$$

2. Bridge: Linear Alignment

Introduce an intermediate term $\mathcal{R}_T(Z_P W)$: evaluate the Target's risk function on Proxy features aligned to Target space via linear map $W$.

Why? $Z_P$ and $Z_T$ live in incomparable spaces. $Z_P W$ maps Proxy into Target space, creating a common ground — so Feature Error and Head Error can be decoupled cleanly.

3. Triangle Inequality

$$|\mathcal{R}_T - \mathcal{R}_P| \;=\; |\mathcal{R}_T(Z_T) - \mathcal{R}_P(Z_P)| \;\le\; \underbrace{|\mathcal{R}_T(Z_T) - \mathcal{R}_T(Z_P W)|}_{\text{Feature Error}} + \underbrace{|\mathcal{R}_T(Z_P W) - \mathcal{R}_P(Z_P)|}_{\text{Head Error}}$$

4. Lipschitz Bound

Feature Error: $K_{feat} \cdot \|Z_T - Z_P W\|_F$

Head Error: $K_{head} \cdot \dfrac{1}{n}\|Z_P(WH_T - H_P)\|_F$
$\quad = K_{head} \cdot \|\Sigma_P^{1/2}(WH_T - H_P)\|_F$   (where $\Sigma_P = \tfrac{1}{n}Z_P^\top Z_P$)

Math Result: Geometric Insights

By solving for the optimal $W$, we achieve an Exact Geometric Decomposition of the feature error using the Procrustes Similarity ($\Omega$):

$$\min_{W} \frac{1}{n} \| Z_T - Z_P W \|_F^2 = \underbrace{(\rho_T - \rho_P)^2}_{\text{Scale Mismatch}} + \underbrace{2\rho_T\rho_P(1 - \Omega(Z_T, Z_P))}_{\text{Shape Mismatch}}$$

Unified Risk Discrepancy Bound

$$|\mathcal{R}_T - \mathcal{R}_P| \le \underbrace{K_{feat} \sqrt{ (\Delta \rho)^2 + 2\rho_T\rho_P(1-\Omega) }}_{\text{Feature Term}} + \underbrace{K_{head} \left\| \Sigma_P^{1/2}(W H_T - H_P) \right\|_F}_{\text{Head Term}}$$

Geometric Insights: The performance gap is bounded by three interpretable quantities:

  • Scale ($\Delta \rho$): Do the feature norms match?
  • Shape ($\Omega$): Is the intrinsic topology of the manifolds isomorphic?
  • Head ($\|\Sigma_P^{1/2}(WH_T - H_P)\|_F$): Do the classifiers agree on the data's active subspace?

Math Discuss: Deep Dive

1. CKA vs. Procrustes ($\Omega$)

CKA uses the Frobenius norm, which provides a looser bound.

Our $\Omega$ relies on the Nuclear Norm ($\|Z_T^\top Z_P\|_*$). This guarantees an exact geometric decomposition for rotational alignment, strictly tightening the theoretical bound.

2. Head Bound

Head Error is bounded by: $\| \Sigma_P^{1/2} (W H_T - H_P) \|_F$

By incorporating the covariance matrix $\Sigma_P$, we project the error onto the active subspace of the data. Mismatches in "empty" directions don't penalize the bound.

3. $\rho$ (Scale)

$\rho$ represents the Root Mean Square (RMS) energy of the representations. Decoupling $\rho$ from $\Omega$ allows us to distinguish between a model that has structurally collapsed vs. one that is just numerically scaled down.

Three Regimes of $W$

$W = I$ (Identity)

Proxy and Target share the same feature space. No rotation needed — only noise.

$$\|Z_T - Z_P\|_F$$

Application: Quantization (FP16 → INT4 / INT2). Bound simplifies to scale collapse + feature distortion.

$W \in O(d)$ (Procrustes)

Find the best orthogonal rotation to align the two spaces. Solved exactly via SVD.

$$W^* = UV^\top, \quad Z_P^\top Z_T = U\Sigma V^\top$$

Application: Model Merging — finding optimal interpolation $\alpha$ across same-size models.

$W \in \mathbb{R}^{d_P \times d_T}$ (General)

Proxy and Target have different dimensions. A general linear map bridges the gap.

$$\min_{W} \|Z_T - Z_P W\|_F^2$$

Application: Cross-Scale Transfer (small → large model) and Continual Learning (Task A → Task B).

Application: Model Merging

Setup

Parameter interpolation with mixing coefficient $\alpha \in [0,1]$:

$$\theta_T(\alpha) = \theta_{T,0} + \alpha\,\Delta_T \quad \text{(Target: large model)}$$ $$\theta_P(\alpha) = \theta_{P,0} + \alpha\,\Delta_P \quad \text{(Proxy: small model)}$$

Goal: Find $\alpha^* = \arg\min_\alpha \mathcal{R}_P(\alpha)$ cheaply on the proxy, then transfer to the target.

Risk Gap at Any $\alpha$

Applying PRISM with linear alignment $W \in \mathbb{R}^{d_P \times d_T}$:

$$|\mathcal{R}_T(\alpha) - \mathcal{R}_P(\alpha)| \;\le\; \underbrace{K_{feat}\sqrt{(\Delta\rho)^2 + 2\rho_T\rho_P(1-\Omega)}}_{\text{Feature Term}} \;+\; \underbrace{K_{head}\left\|\Sigma_P^{1/2}(WH_T - H_P)\right\|_F}_{\text{Head Term}}$$

Geometric Meaning of Each Term

$\Delta\rho$ — Scale Mismatch
Do large and small model features have similar RMS magnitudes at this $\alpha$?

$1 - \Omega$ — Shape Mismatch
Is the task manifold structure preserved across model scales as $\alpha$ varies?

$\|\Sigma_P^{1/2}(WH_T - H_P)\|_F$ — Head Mismatch
Do both classifiers discriminate the same directions in feature space?

Key insight: If the bound is uniformly small across $\alpha$, the optimal $\alpha^*$ found on the cheap proxy safely transfers to the large target.

Application: Quantization Quality Estimation

Idea

$$\theta_F \xrightarrow{\text{quantize}} \theta_Q \quad \text{(e.g., bf16} \to \text{INT4, NF4, Q2)}$$

Target ($F$): Full-precision model — expensive, ground-truth quality.
Proxy ($Q$): Quantized model — cheap to run.

Goal: Predict quality degradation without running expensive benchmarks.

Key Question

Can we estimate quantization degradation purely from the geometric alignment between $Z_F$ and $Z_Q$, on a small unlabeled calibration set?

Quantization: PRISM Bound

Setup

Target = Full Precision ($F$), Proxy = Quantized ($Q$):

$$Z_F = \phi(X;\,\theta_F), \quad Z_Q = \phi(X;\,\theta_Q)$$

$W = I$: Quantization adds numerical noise but no intentional rotation — same feature space.

Head: For some methods (NF4, GPTQ), lm_head stays in FP16, so $H_Q \approx H_F$ and Head Term $\approx 0$. Others (e.g., GGUF) quantize the head too.

Simplified Bound ($W = I$)

Applying PRISM:

$$|\mathcal{R}_F - \mathcal{R}_Q| \lessapprox K_{feat}\!\sqrt{\underbrace{(\rho_F\!-\!\rho_Q)^2}_{\text{Scale Collapse}} + \underbrace{2\rho_F\rho_Q(1\!-\!\Omega)}_{\text{Feature Distortion}}} + \underbrace{K_{head}\|\Sigma_Q^{1/2}(W H_F - H_Q)\|_F}_{\approx\,0 \text{ for some methods}}$$

Degradation estimated purely from geometric statistics!

Geometric Meaning of Each Term

Scale Collapse ($\Delta\rho$)
Does aggressive quantization shrink feature magnitudes?

Feature Distortion ($1-\Omega$)
Is the structural topology of representations preserved after compression?

Head Term
Non-zero if lm_head is quantized (GGUF). Drops to zero for NF4 / GPTQ.

Quantization Methods: Head Term Analysis

GGUF (llama.cpp, Q4_K, Q2_K…)

Method: Round-to-nearest with adaptive mixed-precision (k-quants). Bit budget distributed across layers by importance score.

Head: lm_head is quantized by default (unless explicitly excluded).

$$H_Q \not\approx H_F \;\Rightarrow\; \text{Head Term} \neq 0$$

Must track Head Term in PRISM bound.

NF4 (bitsandbytes / QLoRA)

Method: Normal Float 4-bit — quantization levels derived from the normal distribution, optimized for pre-trained weight statistics. Supports double quantization.

Head: lm_head kept in FP16.

$$H_Q \approx H_F \;\Rightarrow\; \text{Head Term} \approx 0$$

Head Term drops out of the bound.

GPTQ (AutoGPTQ / ExLlama)

Method: Second-order (Hessian-guided) layer-wise quantization. Minimizes reconstruction error using local curvature — more accurate than round-to-nearest at same bit-width.

Head: lm_head kept in FP16 (standard practice).

$$H_Q \approx H_F \;\Rightarrow\; \text{Head Term} \approx 0$$

Head Term drops out of the bound.

Quantization Experiments: GSM8K

Analysis

The Theorical Results: Worst Cases $\to$ Average Cases

Standard Lipschitz or Spectral bounds often assume the worst-case adversarial direction. By integrating the empirical data covariance $\Sigma_P^{1/2}$ into the Head Bound, PRISM shifts the analysis to the average-case behavior on the actual data manifold, making the bound practically tight.

Remove Scale

By extracting $\rho$ from the formulation, the Procrustes Similarity $\Omega \in [0, 1]$ becomes a strictly scale-invariant metric.

This is crucial: A feature vector that is simply halved in magnitude preserves its semantic meaning perfectly ($\Omega=1$), but standard distance metrics would flag it as a massive error. PRISM accurately isolates pure structural shape.

Application: Catastrophic Forgetting

Setup

Fine-tuning on Task A progressively corrupts Task B performance:

$$\theta_0 \xrightarrow{\text{fine-tune on A}} \theta_t \quad (t = \text{training step})$$

Target ($T$): Frozen pre-trained model $\theta_0$ — preserves Task B knowledge.
Proxy ($P$): Evolving model $\theta_t$ — cheap to evaluate mid-training.

$W \in O(d)$: Networks undergo global feature rotations during adaptation — alignment is non-trivial.

Goal: Monitor forgetting continuously without expensive re-evaluation on Task B.

Forgetting Bound

Applying PRISM with orthogonal alignment $W \in O(d)$:

$$\text{Forgetting}(t) \;\le\; \underbrace{K_{feat}\sqrt{(\rho_0\!-\!\rho_t)^2 + 2\rho_0\rho_t(1-\Omega(Z_0,Z_t))}}_{\text{Backbone Drift}} + \underbrace{K_{pred}\left\|\Sigma_t^{1/2}(WH_0 - H_t)\right\|_F}_{\text{Head Drift}}$$

Ω-Regularizer

$$\mathcal{L}_\text{total} = \mathcal{L}_\text{task}(\theta_t) + \lambda \cdot (1 - \Omega(Z_0, Z_t))$$

$1-\Omega$ — Backbone Geometric Drift
In PEFT/LoRA, head is frozen → backbone drift is the primary driver of forgetting.

Ω-Regularizer vs. EWC
More permissive than L2/EWC: allows rotations, only preserves intrinsic relational geometry.

No Task B labels needed
Forgetting is detected from feature geometry alone — no expensive re-evaluation required.

Application: OOD Detection

Setup

Use a lightweight proxy to audit a large model's reliability:

$$W^* = \arg\min_{W \in O(d)} \|Z_T - Z_P W\|_F \quad \text{(fitted on } \mathcal{D}_\text{ID}\text{)}$$

Target ($T$): Large expensive model — predictions are trusted on ID data.
Proxy ($P$): Small lightweight model — cheap to run on any input.

$W \in O(d)$: Alignment map learned once on in-distribution data $\mathcal{D}_\text{ID}$.

Goal: Flag unreliable inputs without access to the target's logits or ground-truth labels.

Geometric Consistency Score

Define a per-sample anomaly score from the alignment residual:

$$s(x) := \|\phi_T(x) - \phi_P(x)\,W^*\|_2$$

In-distribution: $s(x) \approx 0$ — proxy and target agree geometrically.
OOD: $s(x) \gg 0$ — shared semantic structure breaks down, $W^*$ fails to align.

Directly grounded in the PRISM bound: high $s(x)$ implies high prediction risk gap.

No Logits Required
Proxy audits the target purely through feature alignment — no access to target outputs needed.

Theoretically Grounded
Unlike entropy-based methods, the score is bounded by the PRISM risk gap — not a heuristic.

Multi-View Consistency
Anomaly signal comes from geometric inconsistency across representation spaces.

Application: Generalization Forecasting

Setup

Estimate large model performance on the full test set using a small calibration set:

$$\mathcal{D}_\text{cal} \;\ll\; \mathcal{D}_\text{test}, \quad \mathcal{R}_T(\mathcal{D}_\text{test}) \text{ expensive}$$

Target ($T$): Large fine-tuned model — too costly to evaluate on full $\mathcal{D}_\text{test}$.
Proxy ($P$): Lightweight model — cheap to run on all data.

Goal: Forecast $\mathcal{R}_T(\mathcal{D}_\text{test})$ with valid uncertainty quantification.

Control Variate Estimator

$$\hat{\mathcal{R}}_T(\mathcal{D}_\text{test}) = \underbrace{\mathcal{R}_P(\mathcal{D}_\text{test})}_{\text{cheap proxy}} + \underbrace{\bigl(\mathcal{R}_T(\mathcal{D}_\text{cal}) - \mathcal{R}_P(\mathcal{D}_\text{cal})\bigr)}_{\hat{\Delta}: \text{ calibrated geometric bias}}$$

Geometric Validity Gate

On $\mathcal{D}_\text{cal}$, compute $\Omega(Z_T, Z_P)$ and $\|\Sigma_P^{1/2}(WH_T\!-\!H_P)\|_F$:

Both below $\tau$ → proceed; bias $\hat{\Delta}$ is stable (bounded variance).
Either exceeds $\tau$ → reject proxy, fall back to full evaluation.

Systematic Bias, Not Noise
PRISM bounds variance of $\Delta\ell(x)$ via Popoviciu's inequality — the error is predictable, not random.

Active Criterion
Transforms PRISM from a passive metric into a gating condition for sample-efficient evaluation.

Valid UQ
The gate ensures the forecast comes with theoretical uncertainty guarantees, not just a point estimate.

Application: Cross-Family Capability Transfer

Setup

Can a task solved by one model family transfer to another?

$$\text{e.g., Proxy: Llama-3-8B} \;\longrightarrow\; \text{Target: Gemma-7B}$$

Target ($T$): Model from one family (e.g., Gemma) — disjoint representation space.
Proxy ($P$): Model from a different family (e.g., Llama) — different architecture.

$W \in \mathbb{R}^{d_P \times d_T}$: General linear map bridging the two disjoint spaces.

Goal: Predict whether a capability in the proxy will also appear in the target.

Transfer Inequality

$$\mathcal{R}_T \;\le\; \mathcal{R}_P + \underbrace{K\!\left[\sqrt{(\Delta\rho)^2 + 2\rho_T\rho_P(1-\Omega)} + \varepsilon_\text{head}\right]}_{C_\text{geo}}$$

Aligned ($\Omega \to 1$): $C_\text{geo} \to 0$. If proxy solves the task ($\mathcal{R}_P \approx 0$), target is guaranteed low risk.

Misaligned ($\Omega \ll 1$): $C_\text{geo}$ is large. Even if proxy excels, target performance is not guaranteed.

Why Transfer Succeeds
$\Omega \to 1$ means both families structure the data manifold identically — knowledge is geometrically compatible.

Why Transfer Fails
Low $\Omega$ explains why specific capabilities (e.g., coding styles) fail across families despite similar parameter counts.

Diagnosis
If $\mathcal{R}_T \gg \mathcal{R}_P$: high $\Omega$ → head/implementation bug; low $\Omega$ → fundamental topology mismatch.
